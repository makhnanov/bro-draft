# Быстрый старт - Speedtest API

## Проблема: Ошибка "TypeError: Load failed"

Это означает, что API сервер не запущен.

## Решение (выберите один вариант):

### Вариант 1: Автоматический запуск (РЕКОМЕНДУЕТСЯ)

```bash
./start.sh
```

Этот скрипт автоматически:
- Создаст виртуальное окружение Python (если нужно)
- Установит зависимости
- Запустит API сервер
- Запустит Tauri приложение

### Вариант 2: Ручной запуск

**Шаг 1: Настройка (только первый раз)**

```bash
make setup-venv
```

**Шаг 2: Запуск API сервера**

Откройте ПЕРВЫЙ терминал:
```bash
make start-api
```

Вы должны увидеть:
```
 * Running on http://127.0.0.1:5000
```

**Шаг 3: Запуск приложения**

Откройте ВТОРОЙ терминал:
```bash
make start-development
```

## Проверка работы API

```bash
curl http://127.0.0.1:5000/api/health
```

Должен вернуть:
```json
{"status":"ok"}
```

## Важно!

**API сервер должен быть ВСЕГДА запущен** когда работает приложение, иначе speedtest не будет работать.

## Автоматический запуск через Makefile

```bash
make start-all
```

Это запустит API в фоне и затем приложение.

## Остановка

Нажмите `Ctrl+C` в терминале с API сервером или:

```bash
pkill -f speedtest-api.py
```
